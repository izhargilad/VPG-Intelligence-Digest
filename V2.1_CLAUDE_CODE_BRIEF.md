## V2.1 Scope Summary

V2.1 refocuses the platform around two primary dimensions: **Business Unit** and **Industry**. The goal is to build an intelligence capture and analysis engine — not a delivery mechanism.

**What V2.1 does:**
1. Structured signal database organized by BU × Industry (LinkedIn taxonomy)
2. Reddit monitoring as a new signal source
3. Google Trends integration for demand sensing
4. Self-expanding keyword system (auto-adds keywords appearing in 3+ sources)
5. Executive dashboard with BU-centric views, timeframe filtering, and Excel/PPT export
6. Manual on-demand report generation (no automated email delivery required for V2.1)

**What V2.1 does NOT change:**
- V1 weekly email digest remains functional — do NOT disable or break any V1 functionality
- V1 source collection (RSS, web scraping, competitor monitoring) continues unchanged
- V1 validation engine (3-source rule) continues unchanged
- V1 Gmail delivery module continues working

**Key design principles:**
- Every screen, every query, every export starts with BU and Industry as primary filters
- Timeframe selection (last week, last month, last quarter, custom date range) available everywhere
- Dashboard must be exportable to Excel and PowerPoint at any point
- Signal capture is the priority — delivery is manual for now

### 1.3 Industry Assignment Logic

When the AI analysis engine processes a signal, it must:
1. Determine which industry/industries the signal belongs to (from the configured list)
2. Use the industry → BU mapping to assign BUs
3. Allow a signal to belong to multiple industries and multiple BUs
4. Store these relationships in the junction tables (signal_industries, signal_bus)

## 3. Self-Expanding Keyword System

### 3.1 How It Works

After each collection cycle, run a keyword extraction pass:

1. **Extract:** Use Claude Sonnet to identify all named entities from the week's signals: company names, product names, technology terms, geographic locations, regulatory references
2. **Count:** For each extracted entity, count how many independent sources mention it
3. **Threshold:** If an entity appears in **3+ independent sources** AND is not already in the keyword database:
   - Auto-classify its type (technology, competitor, geography, product, company, regulation)
   - Auto-assign relevant BUs and industries based on the context of the signals where it appeared
   - Add to `auto_keywords` table with status "active"
   - Log the discovery in the dashboard for user review
4. **Next collection:** Newly added keywords are included in the next collection cycle's search terms

### 3.2 Keyword Management UI

Add a new tab to the Config UI: **"Keywords"**

- Table view: Keyword, Type, BU tags, Industry tags, Source count, Status, Discovered date
- Filter by: Type, BU, Industry, Status (active/paused/rejected)
- Actions per keyword: Pause (stops monitoring), Reject (removes permanently), Edit (modify tags)
- Manual add: Form to add keywords manually with type, BU, and industry assignments
- **"Recently Discovered" banner** at top showing keywords added in the last collection cycle
- Bulk actions: select multiple → pause, reject, or re-tag

### 3.3 Keyword Feedback Loop

When a keyword is paused or rejected by a user, log this decision. Over time, use the rejection patterns to refine the extraction — e.g., if users consistently reject keywords of type "geography" from certain subreddits, increase the threshold for geographic keywords from Reddit to 5+ sources.

## 4. Timeframe Filtering

### 4.1 Available Timeframes

Every dashboard view and report generation must support these timeframes:

| Option | Date Range |
|--------|-----------|
| Last Week | Monday 00:00 to Sunday 23:59 of the prior week |
| Last Month | 1st to last day of prior calendar month |
| Last Quarter | Standard calendar quarter (Q1 = Jan-Mar, Q2 = Apr-Jun, etc.) |
| Year to Date | January 1 of current year to today |
| Custom Range | User-selected start date and end date (date picker) |

### 4.2 Implementation

- Timeframe selector appears at the top of every dashboard view, above all other filters
- Default: "Last Month"
- Changing the timeframe re-queries all data on the page (signals, trends, charts, stats)
- Export functions use the currently selected timeframe
- API endpoints accept `date_from` and `date_to` query parameters
- Provide helper endpoints or logic for named ranges:
  - `GET /api/signals?timeframe=last_week`
  - `GET /api/signals?timeframe=last_month`
  - `GET /api/signals?timeframe=last_quarter`
  - `GET /api/signals?timeframe=ytd`
  - `GET /api/signals?date_from=2026-01-15&date_to=2026-02-15`

## 12. Critical Rules for V2.1

1. **BU × Industry are the primary dimensions.** Every signal, trend, keyword, and recommendation must be tagged with both. Every dashboard view and export starts with these as filters.
2. **V1 stays functional.** Do not break, disable, or remove any V1 functionality.
3. **LinkedIn industry taxonomy.** Use it consistently. When adding new industries via UI, prompt the user to select from LinkedIn's standard list or explicitly mark as "Custom."
4. **Auto-keywords: Reddit ≠ 3 sources.** Reddit can be 1 of 3 required sources, but at least 2 must be traditional sources.
5. **Timeframe filtering everywhere.** Every view, every API endpoint, every export.
6. **Exports must be presentation-ready.** Excel has proper formatting, filters, and branding. PowerPoint has professional slides that can go directly into a leadership meeting.
7. **Dashboard loads in < 2 seconds.** Use proper database indexes and efficient queries.
8. **Self-expanding keywords need human oversight.** Auto-discovered keywords are visible in the UI immediately. Users can pause or reject before the next collection cycle.
9. **All configuration via UI.** Industries, keywords, subreddits, BU mappings — everything editable without code changes.
