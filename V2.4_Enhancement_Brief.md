## 1. Executive Dashboard — Complete Redesign

### 1.1 Problem

The current Executive Dashboard shows 3 generic KPI cards (Total Scored Signals, Average Score, BUs with Active Signals) and a Competitor Pulse bar chart. A BU head opens this and cannot answer: "What in my markets needs my attention right now?" The view is descriptive, not actionable.

### 1.2 New Design Concept

The dashboard must answer ONE question: **"What does [BU Head] need to act on this [period] across [their industries]?"**

The dashboard has two modes:
- **All-BU Overview** (default when "All BUs" is selected) — a heatmap/summary showing which BUs and industries have the hottest activity
- **Single-BU Deep View** (when a specific BU is selected) — a full strategic briefing for that BU

- The 3×3 BU heatmap grid is the key new element. Use CSS grid, 3 columns.

### 4.3 Scan Behavior — Deduplication Rules

The scan must be **incremental and idempotent**:

1. **URL-based dedup:** If a signal's primary source URL already exists in the database, do NOT create a new signal. Instead, update the existing signal's metadata if the new data is more recent.
2. **Headline similarity dedup:** Before inserting a new signal, check if any existing signal has a headline with >85% string similarity (use fuzzy matching). If found, merge: update source count, add new sources to the existing signal, re-score if new information changes the analysis.
3. **Version tracking:** Add a `version` field to the signals table. Each re-scan that updates an existing signal increments its version. Keep an `updated_at` timestamp.
4. **Scan log:** Record each scan run: `{ scan_id, started_at, completed_at, date_from, date_to, sources_used, signals_new, signals_updated, signals_unchanged, errors }`.

```sql
ALTER TABLE signals ADD COLUMN version INTEGER DEFAULT 1;
ALTER TABLE signals ADD COLUMN first_seen_at DATETIME;

CREATE TABLE scan_log (
  id TEXT PRIMARY KEY,
  started_at DATETIME,
  completed_at DATETIME,
  date_from DATE,
  date_to DATE,
  sources_used JSON,
  signals_new INTEGER DEFAULT 0,
  signals_updated INTEGER DEFAULT 0,
  signals_unchanged INTEGER DEFAULT 0,
  errors JSON,
  status TEXT DEFAULT 'completed'
);
```

### 4.4 Send Digest Behavior

- Composes email from **already-stored signals** (no new collection)
- Filters by date range, BU, Industry, and minimum score
- "Preview Email" opens the HTML in a modal or new tab
- "Send Digest" sends to selected recipients via Gmail API
- Records delivery in the existing digests table

## 5. Reddit — Fix Zero-Signal Issue

### 5.1 Diagnosis

The Reddit monitoring page shows 33 active subreddits with 0 signals across all of them. This indicates the collection pipeline is not executing or not authenticating properly.

### 5.2 Diagnostic Steps (Claude Code should implement)

Create a diagnostic script `scripts/reddit_diagnose.py`:

```python
#!/usr/bin/env python3
"""Reddit collection diagnostic — run this to identify why signals are zero."""

import os
import sys

def check_env():
    """Check if Reddit credentials exist in .env"""
    required = ['REDDIT_CLIENT_ID', 'REDDIT_CLIENT_SECRET', 'REDDIT_USERNAME', 'REDDIT_PASSWORD']
    missing = [k for k in required if not os.environ.get(k)]
    if missing:
        print(f"❌ MISSING ENV VARS: {', '.join(missing)}")
        print("   → Create a Reddit app at https://www.reddit.com/prefs/apps")
        print("   → Choose 'script' type")
        print("   → Add credentials to your .env file")
        return False
    print("✅ All Reddit environment variables present")
    return True

def check_praw():
    """Check if PRAW can authenticate"""
    try:
        import praw
        reddit = praw.Reddit(
            client_id=os.environ['REDDIT_CLIENT_ID'],
            client_secret=os.environ['REDDIT_CLIENT_SECRET'],
            username=os.environ['REDDIT_USERNAME'],
            password=os.environ['REDDIT_PASSWORD'],
            user_agent=os.environ.get('REDDIT_USER_AGENT', 'vpg-intelligence:v2.4')
        )
        # Test authentication
        user = reddit.user.me()
        print(f"✅ PRAW authenticated as: {user.name}")
        return reddit
    except Exception as e:
        print(f"❌ PRAW authentication failed: {e}")
        return None

def check_subreddit_access(reddit):
    """Check if we can actually read from configured subreddits"""
    test_subs = ['robotics', 'AskEngineers', 'manufacturing']
    for sub_name in test_subs:
        try:
            sub = reddit.subreddit(sub_name)
            posts = list(sub.hot(limit=3))
            print(f"✅ r/{sub_name}: accessible, {len(posts)} posts retrieved")
        except Exception as e:
            print(f"❌ r/{sub_name}: {e}")

def check_keyword_matching(reddit):
    """Test if keyword search returns results"""
    try:
        sub = reddit.subreddit('robotics')
        results = list(sub.search('load cell OR force sensor OR torque sensor', limit=5, time_filter='month'))
        print(f"✅ Keyword search in r/robotics: {len(results)} results for 'load cell OR force sensor'")
        for r in results[:3]:
            print(f"   → {r.title[:80]}... (score: {r.score})")
    except Exception as e:
        print(f"❌ Keyword search failed: {e}")

if __name__ == '__main__':
    print("=" * 60)
    print("VPG Reddit Collection Diagnostic")
    print("=" * 60)
    
    if not check_env():
        print("\n⚠️  Fix environment variables first, then re-run.")
        sys.exit(1)
    
    reddit = check_praw()
    if not reddit:
        print("\n⚠️  Fix PRAW authentication first, then re-run.")
        sys.exit(1)
    
    check_subreddit_access(reddit)
    check_keyword_matching(reddit)
    print("\n" + "=" * 60)
    print("Diagnostic complete.")
```

### 5.3 Reddit App Setup Instructions

If credentials are missing, the user needs to:

1. Go to https://www.reddit.com/prefs/apps
2. Click "create another app..." at the bottom
3. Fill in:
   - **name:** vpg-intelligence
   - **type:** select "script"
   - **description:** VPG market intelligence collection
   - **redirect uri:** http://localhost:8080
4. Click "create app"
5. Note down:
   - **client_id:** the string under the app name (looks like: `AbCdEf12345`)
   - **client_secret:** the "secret" field
6. Add to `.env`:
```
REDDIT_CLIENT_ID=AbCdEf12345
REDDIT_CLIENT_SECRET=your_secret_here
REDDIT_USERNAME=your_reddit_username
REDDIT_PASSWORD=your_reddit_password
REDDIT_USER_AGENT=vpg-intelligence:v2.4 (by /u/your_username)
```

### 5.4 Collection Pipeline Fix

After credentials are working, ensure the Reddit collector:

1. **Iterates all active subreddits** from the database/config (not a hardcoded list)
2. **Searches with VPG keywords** using PRAW's `subreddit.search()` with `time_filter='week'`
3. **Also scans hot/new posts** (not just search) — some relevant posts won't match exact keywords
4. **Runs the AI relevance filter** (Claude Sonnet, confidence > 0.7) on each candidate post
5. **Stores qualifying signals** in the signals table with `source_channel = 'reddit'` and corresponding `reddit_signals` record
6. **Logs collection stats** per subreddit (posts scanned, posts filtered, signals created)

Add a "Test Collection" button on the Reddit management page that runs collection for a single subreddit and shows results in real-time (useful for debugging).

### 6.4 "What's Moving" Cards — AI Generation

The top "What's Moving" section is AI-generated after each scan. The prompt:

```
Given the following signals for [BU] collected between [date_from] and [date_to],
identify the top 3-5 NAMED trends. For each trend:

1. Give it a specific name (not "Industry Growth" but "Humanoid Robotics Manufacturing Expansion")
2. Calculate the % change in signal volume vs. prior equal period
3. List the specific companies involved
4. Cite the highest-scored signal headline as evidence
5. Tag the relevant industry
6. Classify as: RISING, DECLINING, NEW, or PERSISTENT

Format as structured JSON.
```

Store the generated trends in a new table:
```sql
CREATE TABLE trend_alerts (
  id TEXT PRIMARY KEY,
  created_at DATETIME,
  bu_code TEXT,
  industry TEXT,
  trend_name TEXT,

  signal_count INTEGER,
  period_weeks INTEGER,
  companies JSON,
  top_signal_id TEXT,
  top_signal_headline TEXT,
```
